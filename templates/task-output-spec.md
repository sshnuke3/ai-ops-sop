# 任务产物清单模板（task-output-spec.md）

> **用途**：给一个长任务（3+ 步）写"阶段 × 产物 × 退出标准"清单。
> **何时写**：任务开始前。哪怕只写 3 阶段也好；不写 = AI 偷懒 / 跳步骤 / 跑飞。
> **抄走方式**：复制本文件 → 改名 `task-output-spec.md` → 按下面 blank 处填空。

---

## 模板（blank，照抄改）

```markdown
# <任务名> 任务产物清单

> 创建日期：YYYY-MM-DD
> 任务一句话描述：<不超过 50 字>
> 任务类型：<代码 / 内容生产 / 数据处理 / 跨工具链 / 其他>

## 阶段清单

### 阶段 1：<阶段名>
- **输入**：<文件 / 链接 / 上游产物路径>
- **产出**：<具体文件路径 + 一句话内容>
- **退出标准**：<机器可校验：退出码 / 文件存在 / 日志关键词 / 截图比对>

### 阶段 2：<阶段名>
- **输入**：<来自阶段 1 的产物，**不允许直接读用户原话**>
- **产出**：<同上>
- **退出标准**：<同上>

### 阶段 3：<阶段名>
- **输入**：<来自阶段 2 的产物>
- **产出**：<同上>
- **退出标准**：<同上>

### 阶段 4（如有）：<阶段名>
- ...

### 阶段 N：提交 / 发布
- **输入**：<所有上游产物>
- **产出**：<git commit / API 调用结果 / 推送完成截图>
- **退出标准**：<commit hash 存在 / API 响应 200 / 文件散落位置确认>

## 不允许的跳步

- ❌ 阶段 3 的输入不能来自阶段 1（必须经过阶段 2）
- ❌ 任何阶段不能"凭印象判断完成"——必须有退出标准验证
- ❌ 同一阶段不能产出多种类型的产物（拆成两个阶段）

## 失败时的回退路径

- 阶段 N 退出标准未达 → 回退到阶段 N-1 重新产出
- 阶段 N 连 2 次失败 → **停下问用户**，不硬试
```

## 填好示例（看完即可抄结构）

### 示例：写一篇 2000 字知乎专栏「AI 长任务为何总卡住」

```markdown
# 知乎专栏「AI 长任务为何总卡住」任务产物清单

> 创建日期：2026-07-21
> 任务一句话描述：写一篇 2000 字知乎专栏，解释 AI 长任务卡住的根因和应对方法
> 任务类型：内容生产

## 阶段清单

### 阶段 1：选题与素材沉淀
- **输入**：用户原话 + 3 篇参考文章链接
- **产出**：`drafts/zhihu-ai-tasks/draft.md`（200-500 字论点摘要 + 5 个一手案例标注）
- **退出标准**：
  - ✅ 文件存在
  - ✅ 至少 3 个案例带具体场景（不是抽象描述）
  - ✅ 论点摘要 < 500 字

### 阶段 2：写正文
- **输入**：`drafts/zhihu-ai-tasks/draft.md`（**不许回到用户原话**）
- **产出**：`drafts/zhihu-ai-tasks/full-text.md`（1500-2200 字）
- **退出标准**：
  - ✅ 字数 1500-2200
  - ✅ 3 个章节标题清晰
  - ✅ 至少 1 处自检清单（哪 3 件事今天就能做）

### 阶段 3：自检与配图
- **输入**：`full-text.md`
- **产出**：`drafts/zhihu-ai-tasks/figures/`（至少 2 张图）+ `full-text.md` 内嵌 `<figure>` 引用
- **退出标准**：
  - ✅ 至少 2 张图存在
  - ✅ `full-text.md` 末尾"自检"小节保留

### 阶段 4：上传与发布
- **输入**：所有上游产物
- **产出**：知乎专栏链接
- **退出标准**：
  - ✅ HTTP 200 响应
  - ✅ 链接可公开访问
  - ✅ 文末自检段还在

## 不允许的跳步

- ❌ 阶段 2 不能直接读用户原话（必须读 stage 1 的 draft.md）
- ❌ 阶段 3 不能跳过图片就直接进 stage 4
- ❌ 阶段 4 不能"凭印象已发送"——必须有 HTTP 响应证据

## 失败时的回退路径

- 阶段 2 字数不达标 → 回阶段 1 补充案例，回到阶段 2 重写（**不**跳阶段 2 重读原话）
- 阶段 4 上传失败 → 重新生成图片 + 检查 markdown 内嵌语法
- 阶段 4 连续 2 次失败 → **停下来找用户，不硬试**
```

## 三个最容易写错的点

| 错 | 正 |
|---|---|
| "产出：一份报告"（模糊） | "产出：`drafts/foo.md`，包含 3 节 + 1 个表格"（具体） |
| "退出标准：看起来 OK 就行" | "退出标准：`wc -l file.md` ≥ 100" 或 "必须有 3 个章节标题" |
| 8 个阶段挤一个段落 | 每个阶段独立一节，**输入字段必须写"来自阶段 N 的产物"** |

---

## 反模式：这些写法都算"没写"

- ❌ 只在脑子里想、没落盘
- ❌ 退出标准写"我会做好的"（人类承诺，不是机器可校验）
- ❌ 跨阶段没有"输入字段"
- ❌ 阶段之间有循环依赖（A 输出给 B，B 又输出给 A）——流水线不能有环
